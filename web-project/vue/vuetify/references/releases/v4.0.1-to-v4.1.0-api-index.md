# Vuetify Release API Index v4.0.1 to v4.1.0

This index captures component and API changes that are worth consulting during implementation. It intentionally omits ordinary visual bug fixes unless they affect API usage, accessibility, SSR, typing, or component selection.

## Core Promotions in v4.1.0

Treat these as core Vuetify capabilities in Vuetify 4.1+ before introducing local wrappers or third-party replacements:

- validation rules
- `VIconBtn`
- `VStepperVertical`
- `VPullToRefresh`
- `VFileUpload`
- `VDateInput`
- `VColorInput`
- `VPicker`

Source: [v4.1.0](./v4.1.0.md)

## New Components and Labs

- `VDateRangePicker` added as a labs component in v4.1.0.
- `VHeatmap` added as a labs component in v4.1.0.
- `VHighlight` added as a labs component in v4.1.0.
- `VMonthPicker` added as a labs component in v4.1.0.
- `VProgress` added as a labs component in v4.0.3.
- `VVideoControls` can be used as a standalone component since v4.0.6.

Sources: [v4.1.0](./v4.1.0.md), [v4.0.6](./v4.0.6.md), [v4.0.3](./v4.0.3.md)

## Data Tables and Tables

- `VDataTable` bottom slot exposes `prevPage`, `nextPage`, and `setPage` since v4.0.2.
- `VDataTable` sortable headers include `aria-sort` since v4.1.0.
- `VDataTable` selection controls include aria labels since v4.1.0.
- `VDataTable` `loading` accepts an object with `side` since v4.1.0.
- `VDataTable` has an `expanded` slot with transition support since v4.1.0.
- `VDataTable` slot props include filtered `itemsLength` since v4.1.0.
- `VDataTable` has a mobile header slot since v4.1.0.
- `VDataTable` group-by state supports `v-model:opened` since v4.1.0.
- `VDataTable` can highlight search matches since v4.1.0.
- `VTable` has a `caption` slot, passes aria attributes to the underlying `<table>`, and supports `gridlines` since v4.1.0.

Sources: [v4.1.0](./v4.1.0.md), [v4.0.2](./v4.0.2.md)

## Inputs, Pickers, and Forms

- `VCalendar` adds `interval-highlight` and 24-hour support in v4.1.0.
- `VDatePicker` supports keyboard arrow navigation in the selection grid since v4.1.0.
- `VColorPicker` adds `hide-input-labels` in v4.1.0.
- `VFileInput` and `VFileUpload` support pasting files and folders since v4.1.0.
- `VFileUpload` adds a `loading` prop in v4.1.0.
- `VNumberInput` adds `grouping` in v4.1.0.
- `VOtpInput` adds groups, accessibility improvements, and emoji support in v4.1.0.
- `VSwitch` adds `size`, square `inset`, and MD3 alignment updates in v4.1.0.
- `VMaskInput` accepts escaped characters in masks since v4.0.3.
- validation rules custom type resolution improved in v4.0.4, before rules were promoted in v4.1.0.

Sources: [v4.1.0](./v4.1.0.md), [v4.0.4](./v4.0.4.md), [v4.0.3](./v4.0.3.md)

## Overlays, Command UI, and Navigation

- `VCommandPalette` adds `closeOnSelect` and `before-select` in v4.0.2.
- `VCommandPalette` and `VOverlay` introduce a viewport location strategy in v4.1.0.
- `VOverlay` fixes make `location` and `origin` practically usable in v4.1.0.
- `VTooltip` adds `color` and hoverable `target="cursor"` support in v4.1.0.
- `useActivator` behavior was fixed in v4.0.5 to avoid closing a hovered menu when a tooltip hides.
- `VList` focus behavior after open was fixed in v4.0.7.

Sources: [v4.1.0](./v4.1.0.md), [v4.0.7](./v4.0.7.md), [v4.0.5](./v4.0.5.md), [v4.0.2](./v4.0.2.md)

## Styling, Theme, Icons, and Utilities

- Material Symbols iconset via UnoCSS was added in v4.0.2.
- `rounded` arbitrary values are supported in v4.1.0.
- `hover-elevation` prop and CSS utilities were added in v4.1.0.
- Theme optional page transitions were added in v4.1.0.
- Theme transparent hex CSS variables were fixed in v4.0.8.
- `VProgressLinear` adds the `split` variant in v4.1.0.
- `VSparkline` adds markers and tooltips in v4.1.0.

Sources: [v4.1.0](./v4.1.0.md), [v4.0.8](./v4.0.8.md), [v4.0.2](./v4.0.2.md)

## Behavior and Compatibility Notes

- `VAvatarGroup` `limit` behavior changed in v4.0.2 to cover the overflow item.
- `VTreeview` binds `indentLines` to item slot in v4.1.0.
- `VTreeview` and `VList` selection logic now wires `value-comparator` in v4.0.7.
- `VVideo` adds `hide-progress-bar` and `src-object` for MediaStream/WebRTC in v4.0.3.
- `VVideo` shows the progress bar by default in v4.0.6.
- SSR rendering with latest Vue was fixed in v4.0.5.
- Type support for `noUncheckedSideEffectImports` was added in v4.0.5.
- `VCol` `offset-*` prop types were corrected in v4.0.4.
- `VProgressLinear` avoids `opacity: NaN` under SSR in v4.0.9.

Sources: [v4.1.0](./v4.1.0.md), [v4.0.9](./v4.0.9.md), [v4.0.7](./v4.0.7.md), [v4.0.6](./v4.0.6.md), [v4.0.5](./v4.0.5.md), [v4.0.4](./v4.0.4.md), [v4.0.3](./v4.0.3.md), [v4.0.2](./v4.0.2.md)
